ITIL(Information Texhnology Infrastructure Library) => ITサービスマネジメントのペストプラクティス書籍の一部

・ITIL成功理由
- ベンダに依存しない
- 規範的ではない
- ベストプラクティスである

・サービス => 顧客に価値を提供する手段
- コアサービス => 顧客が望む基本的な成果を実現するサービス : インターネット速度
- 実現サービス => コアサービスを提供する為に必要なサービス : 一定の通信速度
- 強化サービス => コアサービスをより魅力的にする為の、オプションサービス : 固定IPアドレス割り当て、無線LAN

- サービスオーナ => 特定のITサービスの提供について説明責任
- サービスマネージャ => ITサービスプロバイダ内のマネージャ

・サービスライフサイクル => ITサービスマネジメントをITサービスのライフサイクル(戦略、設計、移行、運用、継続的改善)の視点でまとめたもの
- サービスマネジメント => 顧客に対し、サービスの形で価値を提供する組織の専門的能力の集まり
- ITサービスマネジメント => 特にITにサービスを実施、管理すること

サービスマイ符サイクル
- サービスストラテジ => 方針やサービスの位置付けを明確にし、ITサービスに対する戦略を立てる
- サービスデザイン => サービス戦略に沿って、ITサービスやプロセスを設計
- サービストランジション => 新規および変更されたITサービスを運用へ移行
- サービスオペレーション => 合意されたレベルでITサービスを提供する様に運用管理
- 継続的サービス改善 => より良い戦略、設計、導入、運用を行える様に改善

・サービスプロバイダ
- タイプI - 内部サービスプロバイダ => 事業部門に内在するサービス・プロバイダ
- タイプⅡ - シェアード・サービス部門 => 複数の事業部門に共有ITサービスを提供
- タイプⅢ - 外部サービス・プロバイダ => 外部顧客にITサービスを提供するサービス・プロバイダ

・ITインフラストラクチャ => ITサービス提供に必要なハードウェア・ソフトウェア・ネットワーク・設備など
・サプライヤ => ITサービス提供に必要なITインフラストラクチャやITサービスをITサービス・プロバイダに提供する会社・組織

・プロセス => 特定の達成目標を実現するために設計された一連の活動
-インプットはデータや情報、アウトプットは期待される結果や成果物
・プロセスモデル => プロセス、プロセス・コントロール・プロセスの実現要因
- トリガ => プロセスの活動を開始する要因
プロセスの実現要因 => リソースと能力によって実現されることを意味する
- リソース => ITインフラストラクチャや人材、資金などの資産
- 能力 => ナレッジ(知識)や人材(経験、スキル)、組織などの無形資産を意味する

・プロセスの特性
- 測定可能性 => 効率的に運用できているか
- 特定の結果 => 個別に識別でき、数えられるもの
- 顧客 => 利害関係者に提供、期待に応えなければならない
- 特定のトリガへの応答性 => 特定のトリガに反応してプロセスは開始されるとゆう意味

・プロセスに関係する重要用語
- 機能 => 人材で構成されるチーム・グループと使用するツールやリソース
- 役割 => 個人・チームに与えられた責任・活動・職権。プロセス・機能内で定義
・RACIモデル => 役割定義する際、責任の所在を明確にし、組織の意思決定を支援
-- Responsible:実行責任者 => 業務実行に責任を持つ人(1人・複数)
-- Accountable:説明責任者 => 個々の活動の説明責任者(1人のみ)
-- Consuleted:協議先 => 相談を受け、意見を求められる人々(複数)
-- Informed:報告先 => 常に最新の進捗状況を受け取る人々(複数)
・プロセスの責任者
-- プロセス・オーナ => プロセスの成果に最終的な説明責任を負う
-- プロセス・マネージャ => プロセス運用管理に説明責任を負う
-- プロセス実務者 => 1つ以上のプロセス活動を実施することに責任を負う
